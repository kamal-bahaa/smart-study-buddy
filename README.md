## 🧠 Overview

Smart Study Buddy is a graduation project that helps students study more effectively. Given a lecture PDF, the system automatically generates:

- 🃏 **Flashcards** — Question/answer pairs with difficulty levels (Easy / Medium / Hard), generated by a **fine-tuned LLaMA-3-8B** model trained specifically on educational Q&A data
- 📝 **MCQ Quiz** — Multiple-choice questions with shuffled options and full document coverage, generated by a **base LLaMA-3-8B Instruct** model
- 📄 **Summary** — Structured academic summary broken into main topic, key concepts, important details, and conclusion via **Groq llama-3.3-70b** — exportable as a styled PDF
- 🌐 **Translation** — Real-time Arabic translation of any selected text using **Groq**

All flashcards and MCQs are delivered via **Server-Sent Events (SSE)** so the user sees content appearing progressively in real time.

### PDF Text Extraction Pipeline

```
PDF Upload → SHA-256 Hash Check → PyMuPDF (local extraction) → Groq (text cleaning) → PostgreSQL
```

> If the same PDF was uploaded before (same content, any filename), the server returns the existing document immediately without re-processing.

### Flashcard Generation Pipeline

```
PDF Text → Node.js chunks (2000 chars) → FastAPI sub-chunks (600 chars)
         → Fine-tuned LLaMA-3-8B → Flashcard JSON → SSE Stream → Flutter
```

## 🤖 AI Services

### 1. Flashcard Generation Service
> Runs on Google Colab — URL registered dynamically at startup via `/api/internal/register-service`

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/drive/1cAOVGteesIuHzikbOjOjeEP05LQhPwU_#scrollTo=9VxLUbw4iN9l)

**Model:** Fine-tuned `LLaMA-3-8B` with LoRA adapters trained on custom educational Q&A data

**Key features:**
- Custom Alpaca-format prompt template
- Weighted difficulty distribution: `40% Easy · 40% Medium · 20% Hard`
- 5 question types per difficulty level
- JSON extraction with 2-strategy fallback (full match → field-by-field regex)
- In-session deduplication by question text
- 2× faster inference via [Unsloth](https://github.com/unslothai/unsloth)

**Endpoint:**
```
POST /generate-flashcard
Body: { "context": "...", "total_length": 0 }
```

---

### 2. MCQ Generation Service
> Runs on Google Colab — URL registered dynamically at startup via `/api/internal/register-service`

[![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/drive/1wPq-91nth770P3fo299m53nJFKlbPgXn?authuser=1#scrollTo=ziQFFTPbyjwT)

**Model:** Base `LLaMA-3-8B Instruct` (no fine-tuning)

**Key features:**
- Continuous 100% coverage chunking — zero gaps across the document
- `shuffle_options()` — removes answer bias (correct answer not always B)
- `_normalize_mcq()` — fixes malformed answers like `"A|D"` → `"A"`
- 2-strategy JSON extraction matching the flashcard service
- Separate `/generate-mcqs-chunk` endpoint for SSE streaming

**Endpoints:**
```
POST /generate-mcqs          → Regular generation (full text, internal chunking)
POST /generate-mcqs-chunk    → Streaming generation (pre-chunked by Node, limit=2)
```

---

### 3. PDF Extraction Service
> Runs locally on **port 8000** alongside the Node.js server

**Stack:** PyMuPDF (text extraction) + Groq `llama-3.3-70b-versatile` (cleaning)

**What it does:**
1. Extracts raw text from PDF using PyMuPDF (fast, no API needed)
2. Sends raw text to Groq to remove page numbers, footers, headers, and formatting artifacts
3. Returns clean, readable academic text ready for AI generation

**Endpoint:**
```
POST /extract
Body: multipart/form-data  →  file: <PDF>
```

## ⚙️ Getting Started

### Prerequisites

- Node.js ≥ 20
- PostgreSQL 16
- Python 3.11 (for local AI service)
- Google Colab account (for flashcard & MCQ services)
- ngrok (for exposing the backend)

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/your-username/smart-study-buddy.git
cd smart-study-buddy

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env
# Fill in all required values (see Environment Variables section)

# 4. Run database migrations
npx prisma migrate dev

# 5. Set up the Python AI service virtual environment (first time only)
cd ai-service
python -m venv venv
venv\Scripts\activate       # Windows
pip install -r requirements.txt
cd ..
```

### Running Everything

A single command starts the Node.js server, the Python AI service, and the ngrok tunnel together:

```bash
npm start
```

This uses `concurrently` to run all three processes in one terminal:

| Process | What it runs |
|---|---|
| `[0] ai` | Python FastAPI service on port 8000 |
| `[1] server` | Node.js / Express on port 3000 |
| `[2] tunnel` | ngrok tunnel exposing port 3000 |

### Running the Colab AI Services

1. Open the Flashcard notebook → Run all cells
2. Open the MCQ notebook → Run all cells
3. Each notebook automatically registers its ngrok URL with the backend via `POST /api/internal/register-service` — no manual `.env` update needed

## 🔐 Environment Variables

Create a `.env` file in the root directory:

```env
# ── Server ────────────────────────────────────────
NODE_ENV=development
PORT=3000

# ── Database ──────────────────────────────────────
DATABASE_URL=postgresql://user:password@localhost:5432/smart_study_buddy

# ── JWT ───────────────────────────────────────────
JWT_SECRET=your_jwt_secret_here
JWT_REFRESH_SECRET=your_refresh_secret_here
JWT_EXPIRES_IN=15m
JWT_REFRESH_EXPIRES_IN=7d

# ── Bcrypt ────────────────────────────────────────
BCRYPT_ROUNDS=12

# ── AI Services ───────────────────────────────────
# Note: Flashcard and MCQ URLs are registered dynamically by Colab at startup.
# No need to set them manually.
GEMINI_API_KEY=your_gemini_api_key
GROQ_API_KEY=your_groq_api_key

# ── Internal ──────────────────────────────────────
INTERNAL_SECRET=your_internal_secret   # Must match the secret set in the Colab notebooks
```

## 📡 API Reference

### Authentication

| Method | Endpoint | Auth | Description |
|---|---|:---:|---|
| `POST` | `/api/auth/register` | ❌ | Register new user |
| `POST` | `/api/auth/login` | ❌ | Login and receive tokens |
| `POST` | `/api/auth/refresh` | ❌ | Refresh access token |
| `POST` | `/api/auth/logout` | ✅ | Logout and revoke token |
| `GET` | `/api/auth/me` | ✅ | Get current user profile |

### PDF Documents

| Method | Endpoint | Auth | Description |
|---|---|:---:|---|
| `POST` | `/api/pdfs` | ✅ | Upload PDF — detects duplicates via SHA-256 hash |
| `GET` | `/api/pdfs` | ✅ | List all user documents |
| `GET` | `/api/pdfs/:id` | ✅ | Get document with flashcards, quiz & summary |
| `DELETE` | `/api/pdfs/:id` | ✅ | Delete document (cascades all related data) |

> **Duplicate detection:** If the same PDF content is uploaded again (even with a different filename), the server returns a `409 Conflict` with the original filename.

### Flashcards

| Method | Endpoint | Auth | Description |
|---|---|:---:|---|
| `POST` | `/api/pdfs/:id/flashcards` | ✅ | Generate flashcards (regular JSON) |
| `POST` | `/api/pdfs/:id/flashcards/stream` | ✅ | Generate flashcards **(SSE streaming)** |
| `GET` | `/api/pdfs/:id/flashcards` | ✅ | Retrieve all flashcards for a document |
| `PATCH` | `/api/flashcards/:id` | ✅ | Update a flashcard (front / back / difficulty) |
| `DELETE` | `/api/flashcards/:id` | ✅ | Delete a flashcard |

#### SSE Stream Events (Flashcards)

```
event: start       →  { totalChunks, fileName }
event: flashcards  →  { chunk, totalChunks, flashcards[] }
event: done        →  { document, count }
event: error       →  { message }
```

### Quiz (MCQ)

| Method | Endpoint | Auth | Description |
|---|---|:---:|---|
| `POST` | `/api/pdfs/:id/quiz` | ✅ | Generate quiz (regular JSON) |
| `POST` | `/api/pdfs/:id/quiz/stream` | ✅ | Generate quiz **(SSE streaming)** |
| `GET` | `/api/pdfs/:id/quiz` | ✅ | Retrieve latest quiz |

#### SSE Stream Events (Quiz)

```
event: start  →  { totalChunks, fileName }
event: mcqs   →  { chunk, totalChunks, mcqs[] }
event: done   →  { document, quizId, questionCount }
event: error  →  { message }
```

### Summary

| Method | Endpoint | Auth | Description |
|---|---|:---:|---|
| `POST` | `/api/pdfs/:id/summary` | ✅ | Generate summary |
| `GET` | `/api/pdfs/:id/summary` | ✅ | Retrieve latest summary |
| `GET` | `/api/pdfs/:id/summary/export` | ✅ | Download summary as a styled PDF file |

### Translation

| Method | Endpoint | Auth | Description |
|---|---|:---:|---|
| `POST` | `/api/translate` | ✅ | Translate selected text to Arabic |

**Request:**
```json
{ "text": "Selected text from summary (max 5000 chars)", "targetLang": "ar" }
```

**Response:**
```json
{
  "original": "...",
  "translated": "...",
  "targetLang": "ar"
}
```

### History

| Method | Endpoint | Auth | Description |
|---|---|:---:|---|
| `GET` | `/api/history` | ✅ | Get activity feed (summaries, quizzes, flashcards) |
| `DELETE` | `/api/history/:id?type=QUIZ` | ✅ | Delete a history item by type and resultId |

> **Delete params:** `type` must be `SUMMARY`, `QUIZ`, or `FLASHCARD`. Get both `resultId` and `type` from `GET /api/history`. Deleting a `FLASHCARD` entry removes all flashcards for that PDF.

### Profile

| Method | Endpoint | Auth | Description |
|---|---|:---:|---|
| `GET` | `/api/profile` | ✅ | Get user profile |
| `PATCH` | `/api/profile` | ✅ | Update display name |
| `PATCH` | `/api/profile/password` | ✅ | Change password |
| `DELETE` | `/api/profile` | ✅ | Delete account (cascades everything) |

### Internal

| Method | Endpoint | Auth | Description |
|---|---|:---:|---|
| `POST` | `/api/internal/register-service` | 🔑 | Register Colab AI service URL |

> Protected by `x-internal-secret` header. Called automatically by Colab notebooks at startup.

## 🗄 Database Schema (Overview)

```
User
 ├── PdfDocument []  (hash field for duplicate detection)
 │    ├── Flashcard []
 │    ├── Quiz []
 │    │    └── Question []
 │    └── Summary []
 └── refreshToken
```

All child records are cascade-deleted when a `PdfDocument` is removed.

## 🧪 Testing with Postman

1. **Register** → `POST /api/auth/register`
2. **Login** → `POST /api/auth/login` → copy the `accessToken`
3. Add header: `Authorization: Bearer <accessToken>` to all protected requests
4. **Upload PDF** → `POST /api/pdfs` (form-data, key: `file`)
5. **Stream Flashcards** → `POST /api/pdfs/:id/flashcards/stream`
6. **Export Summary** → `GET /api/pdfs/:id/summary/export` → Save Response as `.pdf`

> For SSE endpoints, use `curl` with the `-N` flag:
> ```bash
> curl -N -X POST http://localhost:3000/api/pdfs/1/flashcards/stream \
>   -H "Authorization: Bearer <token>"
> ```
